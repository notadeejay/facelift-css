Step 1. Prepare MySQL Workbench

Before you query anything, there are a few things you should do.

1. Move to the Navigator section of MySQL and click the > Tables folder in your navigator to view the tables you will be able to access in this Lab.

2. When querying, the sections of MySQL may get a little crowded, and you may want to adjust the size. Click the GIF below to watch and learn how.

3. Now move over to the SQL Query Tab. This is where you will write your queries in response to the questions below. 

You will enter a query for each question directly in MySQL workbench. Once you run your query and get a result, you will use the Knowledge Check below to check your work.  

Instructions to Run Queries

At the top of the SQL Query Tab you will see three lightning bolts. You only need to focus on the first two.

  • The first lightning bolt on the left will run the entire SQL Query Panel. This is preferable if you want to run all the queries at once.
  • The second lightning bolt on the right will only run the query your cursor is on. This is preferable if you want to run one query at a time.

In this Lab, it is recommended you use the second lightning bolt and run one query at a time to check your answers to each question as you go.

            *

NOT Syntax

Getting Stuck?

If you're getting stuck, look through the drop-downs below for some help.

Did You Get an Error Message?

"I keep getting an error message."
Make sure you are:  

  • Using the correct syntax and ending your queries with a semicolon.
  • Spelling the column name(s) and table name correctly.
  • Not typing plain English on the Query Tab without "--". Dash dash signifies a comment, which tells SQL to ignore that line.
Did You Delete Everything?

Don't Panic!

Missing Values in the World Happiness Dataset

  • Instructions
  • Questions (2)

What is the median of the below list of numbers?

[4, 7, 9, 20, 23, 32, 33, 56]

In some cases, missing values are replaced with "NULL" or "NaN," while in other cases you might find empty cells. It's important to note that when you encounter "NULL" or "NaN" values in a spreadsheet the cells are not actually empty.

Zero (0), however, is a real value. Be sure not to accidentally count zeros as missing data. You should also be careful to not replace missing values with zeroes, unless there is a specific reason zeroes would serve as a good substitute for the missing values.
