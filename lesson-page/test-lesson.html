WHERE and Comparison Operators Practice

For each of the following questions, write your query in the SQL query box. Make sure your query ends in a semicolon (;). After you write your query, press Execute to reveal the dataset. Then, write your new query; the result of your new query will appear below your previous query. 

For these questions, you will use the transactions table.

1. Start by entering SELECT * FROM transactions; . Then, press the Execute button to reveal the dataset.

2. Write a query to show all rows in the transactions table where total_cost is less than or equal to $300. Display all the columns. (To see the total_cost column, scroll to the right after you run the query.)

Hint

What keyword do you need to use to specify a condition? Which operator stands for "less than or equal to"?
Look at the examples in this lesson to find the correct syntax. 

Check Your Answer

Syntax:
SELECT
           *
FROM transactions
WHERE total_cost <= 300;​​​​​​

3. Write a query to show all rows in the transactions table with a transaction_id greater than 8. Show only the customer_name and transaction_id fields. 

Hint

Which operator stands for "greater than"? How do you specify which columns you want to display?

Check Your Answer

Syntax:
SELECT
            customer_name,
            transaction_id
FROM transactions
WHERE transaction_id > 8;

In some cases, missing values are replaced with "NULL" or "NaN," while in other cases you might find empty cells. It's important to note that when you encounter "NULL" or "NaN" values in a spreadsheet the cells are not actually empty.

Zero (0), however, is a real value. Be sure not to accidentally count zeros as missing data. You should also be careful to not replace missing values with zeroes, unless there is a specific reason zeroes would serve as a good substitute for the missing values.
